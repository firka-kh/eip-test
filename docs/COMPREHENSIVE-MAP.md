# Полная карта проекта eip-test / Comprehensive Project Map

Этот документ является «точкой входа» для понимания архитектуры, навигации по коду и быстрого поиска нужных функций.

---

## 📂 Структура файлов и папок

### 1. Корень проекта
- `grant.html` — Единственная точка входа (SPA). Содержит основной HTML-каркас и модальные окна.
- `docs/` — Вся документация (ТЗ, архитектура, история).

### 2. `/src/js/` (Логика)
- `app.js` — Загрузчик: последовательно инициализирует модули и скрывает лоадер.
- **`core/`**
    - `utils.js` — Все утилиты: `AppNotify` (уведомления), `sanitizeText`, `addLog` (аудит), работа с датами и именами файлов.
- **`state/`**
    - `seed-dictionaries.js` — Словари бенефициаров (mockDatabase), базы поиска.
    - `seed-applications.js` — Предзагруженные заявки и протоколы (СП-9001...).
    - `seed-monitoring.js` — Данные визитов мониторинга.
    - `seed-data.js` — Финальная сборка `window.state` и миграции (удаление PIU/ГРП).
    - `store.js` — API доступа к данным: `getApp`, `filterApps`, `checkBeneficiaryDataComplete`.
- **`ui/`**
    - `render.js` — Сердце проекта: отрисовка карточек, таблиц, фильтров, базовая логика переключения режимов.
- **`features/`** (Бизнес-логика этапов)
    - `facilitator.js` — Создание заявок, доработка, работа с черновиком и сканом договора.
    - `gmc.js` — Опросник ШИГ/КУГ, проверка полноты данных, подготовка к реестру.
    - `committee.js` — Групповое рассмотрение заявок, экспорт протоколов.
    - `monitoring.js` — Логика визитов, планирование и заполнение отчетов.

---

## 🧠 Глобальные объекты (Memory & Context)

Все данные хранятся в `window`:
- `window.state.applications` — Все заявки системы.
- `window.state.protocols` — Утвержденные списки Комитета.
- `window.activeMainFilter` — Текущий активный режим (`facilitator`, `gmc`, `committee`, `approved_registry`).
- `window.AppNotify` — Объект для вызова унифицированных уведомлений (`toast`, `confirm`).

---

## 🛠 Ключевые функции (Быстрый поиск)

### Работа с заявками
- `window.getApp(id)` — Получить объект заявки по ID.
- `window.filterApps(statusArray)` — Отфильтровать заявки по статусам.
- `window.addLog(app, actor, actionTj, actionRu, color, icon)` — Записать действие в историю.

### Интерфейс
- `window.renderAllCards()` — Полная перерисовка текущего дашборда.
- `window.updateAllBadges()` — Обновление всех счетчиков и индикаторов.

### Договор (Contract)
- `src/js/ui/render.js` → `renderGrantContractDraftPanel()` — Отрисовка формы черновика.
- `src/js/core/utils.js` → `ensureGrantContractDraft(app)` — Инициализация объекта договора.

## 🔄 Основные бизнес-потоки

1.  **Жизненный цикл:** `draft` → `gmc_review` → `gmc_preparation` → `com_review` → `approved`.
2.  **Доработка:** Если Комитет отклоняет 3-й раз — статус `postponed` (3 мес.). Разблокировка только вручную Фасилитатором.
3.  **Финансы (Молия):** Режим `approved_registry` показывает одновременно **списки (протоколы)** и **заявки**. Поиск по ФИО/ID переключает вид на детали заявителей.
